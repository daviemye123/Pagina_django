Django
Tu mejor framework de desarrollo web

¿QUÉ ES DJANGO?

Es un framework usado para desarrollo web de alto nivel, escrito en Python, que facilita la creación de aplicaciones robustas.

Definición y Filosofía

Django es un framework web de código abierto y alto nivel escrito en Python que fomenta el desarrollo rápido y un diseño limpio y pragmático, bajo la filosofía **"Don't Repeat Yourself (DRY)"**.

Características Principales

Framework Full-Stack

Incluye todo lo necesario para construir aplicaciones web completas sin necesidad de herramientas externas.

Batteries Included

Viene con muchas funcionalidades integradas como autenticación, ORM, admin panel y más.

ORM Potente

Mapeo objeto-relacional que abstrae las consultas SQL de manera elegante.

Admin Automático

Panel de administración generado automáticamente y personalizable.

Sistema de Templates

Separación clara entre lógica y presentación con un motor de plantillas robusto.

Seguridad

Protección integrada contra las vulnerabilidades web más comunes.

FUNCIÓN DE DJANGO

Arquitectura Modelo-Vista-Plantilla (MTV) para desarrollo eficiente

El Patrón de Diseño MTV

Django implementa el patrón de diseño MTV (Model-Template-View), una variación del MVC tradicional, optimizado para el flujo de trabajo de Python.

Componentes Clave

MODEL (Modelo)

Definen la estructura de datos y se comunican con la base de datos mediante el **ORM**. Representan las tablas de tu base de datos.

VIEW (Vista)

Contienen la **lógica de negocio** y procesan las peticiones HTTP. Son el cerebro de tu aplicación, decidiendo qué datos se necesitan.

TEMPLATE (Plantilla)

Archivos HTML con sintaxis especial de Django para renderizar datos dinámicos. Controlan la **presentación visual** para el usuario.

Flujo de Petición en Django

  • El usuario realiza una petición HTTP a una URL.
  • El archivo **`urls.py`** enruta la petición a la **View** (Vista) correspondiente.
  • La View interactúa con el **Model** (Modelo) para obtener o modificar datos en la base de datos.
  • La View pasa los datos al **Template** (Plantilla) para su renderizado final.
  • Se devuelve una respuesta HTTP al usuario con el contenido generado.

ESTRUCTURA DE DJANGO

Organización modular y escalable por proyecto y aplicaciones

Componentes Clave y su Función

settings.py

Archivo central para la configuración de la **base de datos**, aplicaciones instaladas y **seguridad**.

urls.py

Define las **rutas** y dirige el tráfico web (URL) hacia la función de vista (View) adecuada.

manage.py

Utilidad de línea de comandos para tareas como iniciar el servidor, ejecutar migraciones o crear superusuarios.

migrations/

Registra y aplica los **cambios en la estructura de la base de datos** a partir de los `models.py`.

DESVENTAJAS DE DJANGO

Consideraciones antes de usar Django

Puntos a Considerar

CURVA DE APRENDIZAJE

Puede ser complejo para principiantes debido a la gran cantidad de características y convenciones establecidas.

OVERHEAD EN PROYECTOS SIMPLES

Su estructura completa resulta "pesada" para proyectos muy pequeños o prototipos que solo necesitan una función simple (donde Flask sería mejor).

MONOLÍTICO

Incluye muchas funcionalidades que podrías no necesitar, aumentando la complejidad del proyecto inicial.

ACOPLAMIENTO AL ORM

El uso intensivo del ORM puede crear una fuerte dependencia con Django, dificultando la migración a otros ORM o bases de datos.

OPINIONADO

Impone una estructura y convenciones específicas, lo que reduce la flexibilidad comparado con frameworks no opinionados.

NO REAL-TIME NATIVO

Necesita una librería adicional (**Channels**) para funcionalidades en tiempo real como WebSockets.
